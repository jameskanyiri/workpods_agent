# Milestone Workflows

## Phase to milestone

1. Read current project detail.
2. Decide whether the user's phrase describes a phase, checkpoint, or deliverable.
3. Draft a milestone with `start_date` and/or `end_date` (YYYY-MM-DD; see `api.md` for the field schema — there is no `target_date` field) and definition of done.
4. Identify existing tasks to link or starter tasks to propose.
5. Confirm.
6. Create the milestone and, if needed, hand off to task work.

## Milestone risk review

1. Read project detail and milestone statuses.
2. Group tasks by milestone.
3. Identify overdue, blocked, or missing tasks.
4. Summarize milestone risk first, then recommend the single best next move.
