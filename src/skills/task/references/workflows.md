# Task Workflows

## Starter task creation

1. Resolve workspace and project.
2. Read statuses, members, labels, and current tasks.
3. Break the goal into atomic tasks.
4. Check duplicates.
5. Present the breakdown and confirm.
6. Create tasks.
7. Apply assignees and labels.
8. Verify milestone linkage and due dates when relevant.

## Bulk task changes

1. Read current tasks first.
2. Identify the exact affected tasks.
3. Present the delta, not a vague summary.
4. Confirm.
5. Execute updates.
6. Re-read if the user needs a verified state report.
