# Project Execution Workflows

## Move a project forward

1. Read the current project state.
2. Identify the highest-leverage missing structure.
3. If no project exists, hand off to `project`.
4. If the phase should be a milestone, hand off to `milestone`.
5. If milestone work needs decomposition, hand off to `task`.
6. If visible progress should be recorded, hand off to `project-updates`.
7. Verify the resulting state and recommend the next move.
