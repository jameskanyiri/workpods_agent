# Label Workflows

## Create a label strategy

1. Read existing categories and labels.
2. Identify the operational problem:
   filtering, risk tracking, site grouping, team ownership, or client stream.
3. Decide whether status or priority already solves it.
4. If not, propose the minimum useful categories and labels.
5. Confirm.
6. Create them and document their intended use.

## Apply labels to tasks

1. Read the target task and current labels.
2. Apply only the labels that materially improve filtering or reporting.
3. Verify the task shows the applied label when that matters downstream.
